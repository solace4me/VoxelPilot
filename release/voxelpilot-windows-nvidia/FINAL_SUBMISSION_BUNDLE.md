# VoxelPilot Final Submission Bundle

Use this guide to decide what to open first based on who is looking at the project.

## For Judges

Start here:

- `JUDGE_SUMMARY.md`

Then, if they want more context:

- `README.md`
- `DEMO_PRESENTATION.md`
- `PITCH_DECK_OUTLINE.md`

Purpose:

- `JUDGE_SUMMARY.md`
  Best one-page overview of the problem, solution, features, selling points, and roadmap.
- `README.md`
  Best package overview and high-level orientation.
- `DEMO_PRESENTATION.md`
  Best if they want a polished markdown slide version.
- `PITCH_DECK_OUTLINE.md`
  Best if they want a quick structured slide outline.

## For Live Demo Use

Start here:

- `Run_VoxelPilot.bat`

Then keep these nearby:

- `FINAL_RELEASE_CHECKLIST.md`
- `FIRST_RUN_CHECKLIST.md`
- `SAMPLE_WORKFLOW.md`
- `PRESENTATION_SCRIPT.md`

Purpose:

- `Run_VoxelPilot.bat`
  Launches the demo build on the NVIDIA laptop.
- `FINAL_RELEASE_CHECKLIST.md`
  Best last-minute runbook before presenting.
- `FIRST_RUN_CHECKLIST.md`
  Best for first laptop startup validation.
- `SAMPLE_WORKFLOW.md`
  Best for the actual in-app demonstration flow.
- `PRESENTATION_SCRIPT.md`
  Best speaking guide during the live demo.

## For Technical Reviewers

Start here:

- `README.md`
- `JUDGE_SUMMARY.md`

Then, if needed:

- `RUNTIME_NOTES.txt`
- `PITCH_DECK_OUTLINE.md`

Purpose:

- `README.md`
  Good overview of the packaged build and its contents.
- `JUDGE_SUMMARY.md`
  Clear explanation of why the project matters.
- `RUNTIME_NOTES.txt`
  Good for hardware/runtime expectations and troubleshooting.
