VoxelPilot Runtime Notes
===========================

What this app needs
-------------------
- Windows
- An NVIDIA GPU
- A working NVIDIA graphics driver
- OpenGL support on the active display adapter

What to launch
--------------
- Use: Run_VoxelPilot.bat
- First startup guide: FIRST_RUN_CHECKLIST.md
- Guided demo flow: SAMPLE_WORKFLOW.md

What the launcher does
----------------------
- Confirms the app exe exists
- Checks whether nvidia-smi is available
- Adds the CUDA toolkit bin folder to PATH from CUDA_PATH or the newest default CUDA install folder
- Starts the standalone app
- Prints a simple checklist if startup fails

Important behavior
------------------
- This app can compile on systems without an NVIDIA GPU, but it cannot run there.
- If CUDA is unavailable, startup should fail quickly with a clear CUDA runtime message.

Recommended laptop checks
-------------------------
1. Open Command Prompt.
2. Run: nvidia-smi
3. Confirm the NVIDIA GPU and driver are listed.
4. Launch Run_VoxelPilot.bat

If startup fails on the NVIDIA laptop
-------------------------------------
1. Update or reinstall the NVIDIA driver.
2. Make sure the app is using the NVIDIA GPU, not an integrated-only path.
3. Confirm OpenGL works correctly on that machine.
4. If CUDA toolkit is installed, confirm CUDA_PATH is set or keep the default NVIDIA toolkit install location.

Volume data expectations
------------------------
- Headerless raw volume files are supported.
- Supported voxel types: uint8, uint16, float32.
- The import controls now use Width, Height, Depth, and Data Type.
- The selected dataset size must exactly match width x height x depth x bytes_per_voxel.
- Filenames like foot_256x256x256_uint8.raw can be auto-detected by the importer.
